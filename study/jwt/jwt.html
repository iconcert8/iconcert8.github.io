JWT

유저를 인증하기 위한 토큰기반의 도구이다. Statless 서버를 지향하는 요즘 JWT를 활용해 사용자 인증을 한다.

요청 헤더에 발급받은 토큰을 포함시킴으로써 인증을 할 수 있다.

토큰 구조

Header, Payload, Signature를 가지는 구조이며, 기본적으로 BASE64로 인코딩한 토큰이 발급된다.

token = BASE64(Header).BASE64(Payload).BASE64(Signature)

*Signature도 가공된 값이다.
암호화 알고리즘(HMACSHA256)은 예시이다.

Signature = HMACSHA256(BASE64(Header).BASE64(payload), secret-key)


Refresh Token

토큰이 탈취당하면 대처가 어려워진다.

그래서 나온 방법이 Access Token(기본 토큰)의 만료시간을 짧게하여 Access Token이 탈취 당하더라도 금방 사용을 못하게 만든다.

만료시간이 짧아져 재인증 간격이 너무 잦기에 Refresh Token을 추가하였다. Access Token 발급시 Refresh Token을 같이 발급하고, Access Token이 만료되면 Refresh Token을 이용해 Access Token을 재발급 받는다.
