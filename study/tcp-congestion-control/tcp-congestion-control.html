TCP Congestion Control(TCP 혼잡 제어)

한 라우터에 패킷이 몰릴 경우 처리를 할 수 없다. 이를 위해 제어가 필요하다.

AIMD(Additive Increase, Muticative Decrease)
패킷을 1씩 늘려가며 전송, 문제 생기면 절반으로 줄여 전송한다.

Slow Start
패킷을 2배로 늘려가며 전송한다. 혼잡 시 window size 1로 변환한다. 이후 절반까지 2배 증가하고 이후 1씩 증가한다.

Fast Retransmit
손실 시 중복되는 패킷 순번이 생기고 이를 3개정도 받으면 window size를 줄인다.

Fast Recovery
혼잡 시 반으로 줄이고 이후 선형 증가한다.
