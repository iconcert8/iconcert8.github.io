Storage Engine

데이터베이스에 데이터를 삽입/추출/수정/삭제 하는 모듈이다. MySQL(MariaDB)에는 몇가지 Stroage Engine이 존재하고, 이들의 특징을 알아보고자 한다.
MyISAM(마이아이삼)

- MySQL 5.5 이전까지 기본엔진이다.
- 트랜잭션을 지원하지 않는다.
- Table Level Lock, 때문에 일정이상 코어를 늘려도 성능향상이 되지 않는다.
- B-Tree, R-Tree, Full-Text 인덱스 지원
*R-Tree: 2차원을 저장. 공간정보 인덱싱 및 저장에 용이
*Full-Text: 텍스트를 토큰화하여 단어들이 인덱싱되어 있어서 단어로 데이터를 찾을 수 있게 한다.(InnoDB 5.7부터 지원)
- 왜래키 기능이 없다.
- 구조가 단순하다. 성능이 좋다, 특히 조회가 빨라서 읽기 작업에 적합하다.
- 인덱스만 캐싱하고, 데이터는 캐싱하지 않는다.

InnoDB(이노디비)

- MySQL 5.5 부터 기본엔진이다.
- 트랜잭션을 지원한다. commit, rollback, 정합성의 개념이 생겼다.
- 64TB 저장 한계가 있다. 2³²의 tablespace를 지원하고 기본 page size가 16KB이기 때문에 이를 곱하면 64TB이다.
- Row Level Lock, 코어 개수 늘면 성능 향상.
- 왜래키 지원
- 인덱스, 데이터 모두 캐싱 지원
- 구조가 복잡, 많은 공간이 필요하다.
- 클러스터 인덱스 지원, Order by와 같은 정렬 유리.
