ExecutorService

Java에서 스레드 풀의 사용을 효과적으로 돕는 라이브러리이다. Java 5부터 도입되었다. 스레드 풀은 큐에 작업을 담아두고 실행가능한 스레드가 큐에서 작업을 가져와 실행하는 형태를 말한다.

일반적으로 잦은 스레드 생성/해제는 오버헤드가 있어 스레드 풀의 사용이 효과적이다. 그러나 스레드 풀 관리는 손이 많이 가는 작업이다. 스레드 생성 개수 제한, 스레드 라이프사이클(생성, 종료), 스레드 스케쥴링과 같이 많은 고려가 필요하다.

ExecutorService는 스레드의 생성/관리/실행/해제 역할을 모두 맡는다. 작업자는 스레드에서 실행 할 함수에만 집중하여 개발하면 된다.

기본 제공 스레드 풀

Executors.newFixedThreadPool(int n)

n개의 스레드로 고정되어 있는 스레드 풀

Executors.newCachedThreadPool

동적으로 필요한 만큼 스레드가 생성된다. 60초간 작업이 없으면 pool에서 스레드가 제거된다.

Executors.newSingleThreadExecutor

1개의 스레드가 생성된다.

Executors.newScheduledThreadPool

일정 시간 뒤에 작업을 하도록 해주는 스레드 풀이다. 타이머 기능을 스레드 풀에 넣어 놓은 것이다.
