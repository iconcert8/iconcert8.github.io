Context Switching 이란?

메모리에 Task(Process/Thread)를 바꿔가며 실행하는 것이다. 그렇지 않으면 동시다발적인 프로그램 실행(Multitasking)이 불가능하다.

TCB(Thread Control Block) 스위칭이 PCB(Process Control Block) 보다 저렴하다. TCB는 Stack영역을 제외한 메모리영역을 공유하기 때문에 Stack 영역만 스위칭한다.

언제 Context Switching이 발생하는가?

Interrupt가 발생할 때 Context Switching이 일어난다.

  I/O request: 입출력 요청
  Time slice expired: CPU 사용시간 만료
  Fork a child: 자식 프로세스 생성할 때
  Wait for an interrupt: 인터럽트 처리 기다릴 때
