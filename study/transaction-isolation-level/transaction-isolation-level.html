트랜잭션 고립 수준

트랜잭션의 독립성, 고립성을 유지하기 위한 데이터 접근 수준이다.

접근 수준이 낮으면 다음과 같은 문제가 발생한다.

1) Dirty Read
커밋되지 않은 데이터를 읽을 수 있다.

2) Non-Repeatable Read
반복해서 데이터를 읽는 중간에 다른 트랜잭션이 데이터를 수정하여 처음 읽은 데이터와 값이 달라진다.

3) Phantom Read
반복해서 데이터를 읽는 중간에 다른 트랜잭션이 데이터를 삽입하여 없던 데이터가 생기는 상황이다.

고립 수준으로 위에서 생기는 문제들을 차례대로 해결 가능하다.

1) Read Uncommited(Level 0)
커밋되지 않은 데이터를 읽을 수 있다.

2) Read Commited(Level 1)
커밋된 데이터만 읽을 수 있다. Dirty Read 해결.

3) Repeatable Read(Level 2)
선행 트랜잭션 읽기 종료 전까지 갱신/삭제 트랜잭션 개입 불가. Non-Reapeatable Read 해결.

4) Serializable(Level 3)
선행 트랜잭션 읽기 종료 전까지 갱신/삭제/삽입 트랜잭션 개입 불가. Phantom Read 해결.
