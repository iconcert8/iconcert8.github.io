Java 8

JVM Method Area/Permanent Gen 삭제 Java 8 이후 해당영역은 없어지고 class, interface, method 정보는 Metaspace영역으로 옮겨갔다. static field, String pool은 Heap 영역으로 옮겨갔다. 🏗️JVM 그리고 구조

Parallel GC default GC로 Parallel GC를 사용한다. 🧹JVM GC 알고리즘

Stream collection에 담겨 있는 element를 내부 순회하는 코드 패턴, 사용 전까지 element의 계산을 미룬다. 함수형과 결합되어 선언형으로 코드가 이루어진다.

Lambda 함수를 선언 없이 단순히 구현하고 사용하는 형태이다. 익명함수를 간략화 한다.

Instant 기존 Date는 몇가지 문제가 있었다. 1)가변객체, 2)윤초 고려치 않음, 3)월(month)을 0부터 표기, 4)2038년 이후 표기날짜 제한. 문제를 해결하고자 새로운 날짜 Util을 제공하게 된다.

default 인터페이스에는 구현코드가 들어가면 안된다. 그러나 default 메소드는 이를 가능케 한다. default 메소드의 목적은 코드 확장성에 있다.
 기존 인터페이스에 기능을 추가하려면 추상 메소드가 추가되고 하위 구현 클래스들은 강제로 메소드를 오버라이드 해야 했다. 이는 인터페이스가 기능 확장에 매우 닫혀있게 만들었다.
 default 메소드는 인터페이스에 메소드를 구현할 수 있게 한다. 그리고 하위 구현 클래스들은 아무런 작업이 필요없다. 단지 해당 메소드의 기능이 구현클래스에 맞지 않으면 오버라이드 하면된다.

메소드 참조(::) 파라미터가 생략되는 함수 사용 형태이다.
람다의 경우 (str)->System.out.println(str) 로 표현되었다면,
메소드 참조는 System.out::println 로 사용된다.

Optional Nullable 인자를 감싸주는 클래스이다. null 체크를 도와주고 처리도 간단히 할 수 있게 도와준다. 무엇보다 nullable 하다라는 인지를 시켜주기 때문에 코드의 시인성과 안정성을 높일 수 있다.

String 메소드 추가
strip() - 문자열 앞, 뒤 공백제거. 기존 trim()이 제거하지 못하는 유니코드의 공백이 있었고 이를 보완하였다.
stripLeading() - 앞 공백제거
stripTrailing() - 뒤 공백제거
isBlank() - 비어있거나, 공백문자 포함 확인. 기존 isEmpty()는 공백도 문자가 있는것으로 판별.
repeat(int n) - n번 반복 문자열 반환

nio.file.Files 메소드 추가
writeString - 파일에 문자열 쓰기
readString - 파일 문자열 읽기
isSameFile - path가 같은 파일을 가리키면 true

Lambda 'var' 변수 Lambda에 var 로컬변수가 사용 가능하다.

Java 파일 실행 java 파일을 컴파일 안하고 실행이 가능하다.

Http Client 지원 Http Client를 정식으로 도입했다. 전체적인 API 성능향상이 있다.
