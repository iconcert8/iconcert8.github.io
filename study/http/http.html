🌏HTTP의 변천사

iconcert8's log

2023-02-03

HTTP/0.9

원래 버전이 따로 없었지만, 차후 1.0버전과 구분 짓기 위해 0.9를 붙였다.

원 라인 프로토콜이라고 불리며, 요청이 단일 라인으로 구성되었다.
리소스 경로로 요청 가능한 메소드는 GET이 유일하다.

응답 또한 단순하게 파일 내용으로 구성되었다. Http 헤더가 없었고, html파일 이외의 형식은 지원하지 않았다.

HTTP/1.0

응답에 상태코드가 붙게 되었다. 성공/실패를 알 수 있게 된 것이다.

HTTP 헤더가 생겼고 메타데이터로 인한 확장성을 높이는 계기가 되었다. HTML 파일외의 문서가 전송 가능하게 되었다(content-type).

HTTP/1.1

일회성인 커넥션의 오버헤드가 심했다. 때문에 커넥션을 재사용 할 수 있게 하였다.

파이프라이닝 개념을 도입하였다. 첫번째 요청에 응답이 완료되기 전에 두번째 응답을 가능케 하였다.

청크된 응답도 지원하게 된다.

HTTP/2

텍스트 기반의 프로토콜에서 이진(binary)기반의 프로토콜로 변화했다.

병렬 요청이 동일한 커넥션에서 다루어 지게 되었다. 순서를 제거해 순차적으로 응답하는 데서 오는 병목현상을 없앴다.

데이터들 사이의 존재하는 중복(메타정보) 오버헤드를 제거하고, 연속된 요청의 유사한 헤더들은 압축시켰다.

서버 푸시 개념을 도입하였다. 필요하게 될 데이터를 서버에서 임의로 클라이언트에게 전송할 수 있게되었다.

이후 HTTP

Clinet Hints의 도입, 클라이언트가 요구사항/서버 하드웨어 제약사항에 관한 정보를 사전에 미리 주고 받을 수 있게 되었다.

Cookie 내에 보안 관련 접두사를 도입하였다. 쿠키가 변경되지 않았음을 보장하게 된다.
